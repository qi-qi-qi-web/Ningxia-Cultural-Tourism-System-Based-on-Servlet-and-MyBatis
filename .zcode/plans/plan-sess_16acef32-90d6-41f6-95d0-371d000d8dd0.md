# 游玩攻略关联景区功能 - 实现计划

## 概述
为"游玩攻略"功能添加景区关联能力：在景区详情页点击"游玩攻略"显示该景区相关的已发布攻略，发布攻略时可选择关联景区。

---

## 一、数据库变更

### `tourism_db_merged.sql`
在 `travel_guide` 表中新增 `scenic_spot_id` 列：
```sql
ALTER TABLE travel_guide ADD COLUMN scenic_spot_id BIGINT DEFAULT NULL 
  COMMENT '关联景区ID' AFTER user_id,
ADD INDEX idx_scenic_spot (scenic_spot_id),
ADD CONSTRAINT fk_guide_scenic FOREIGN KEY (scenic_spot_id) 
  REFERENCES scenic_spot(id) ON DELETE SET NULL;
```

---

## 二、后端变更

### 1. `TravelGuideMapper.xml` - 5处修改
- **resultMap**: 添加 `<result property="scenicSpotId" column="scenic_spot_id"/>`
- **INSERT SQL**: 字段列表加上 `scenic_spot_id`，值加上 `#{scenicSpotId}`
- **UPDATE SQL**: SET 子句加上 `scenic_spot_id = #{scenicSpotId}`
- **新增查询方法**: `findByScenicSpotId` — 查询某景区下已发布的攻略（JOIN user，status=PUBLISHED）

### 2. `TravelGuideMapper.java`
- 新增方法: `List<TravelGuide> findByScenicSpotId(Long scenicSpotId)`

### 3. `GuideServlet.java` - 3处修改
- **handlePublish()**: 解析 `scenicSpotId` 参数，设置到 TravelGuide 对象
- **新增 action "byScenic"**: 接收 `scenicSpotId` 参数，返回该景区下的已发布攻略 JSON 列表
- **guideToJson()**: JSON 输出中加入 `scenicSpotId` 字段

### 4. `AdminStrategyServlet.java` - 1处修改
- **saveStrategy()**: 解析 `scenicSpotId` 参数，设置到 TravelGuide 对象（与 create/update 两路径都要处理）

## 三、前端变更

### 1. `ScenicService-detail.jsp` - 替换硬编码攻略Tab
- **替换** 第124-143行（硬编码的游玩路线）为动态内容区
- 页面加载时通过 AJAX 调用 `/guide?action=byScenic&scenicSpotId=${scenic.id}` 获取攻略列表
- 有攻略时：展示攻略卡片列表（封面、标题、摘要、作者、浏览量），点击可跳转到 `TravelGuide-detail.jsp?id=X`
- 无攻略时：显示 "对不起，该景区暂时没有游玩攻略"
- 卡片样式与 TravelGuide.jsp 中的攻略卡片保持一致

### 2. `TravelGuide.jsp` - 发布弹窗新增景区选择
- 在标签选择区域下方添加景区下拉选择框（`<select id="guide-scenic">`）
- 页面加载时 AJAX 获取开放景区列表：已有 `ScenicService.jsp` 中通过 scriptlet 查询 `ScenicSpotMapper.findOpen()` 的模式，可参考或用 `/scenic` 相关接口
- 默认选项："不关联景区（通用攻略）"，value 为空
- `submitGuide()` 函数中追加 `formData.append('scenicSpotId', ...)`

### 3. `PersonalCenter.jsp` - PC端发布弹窗新增景区选择
- 同样在 `#pc-publish-guide-modal` 中添加景区下拉框
- `loadPCTags()` 调用时同时加载景区列表
- `submitGuidePC()` 函数中追加 `scenicSpotId`
- `openPublishGuideModalPC(id)` 编辑时预选已关联的景区

### 4. `Admin-strategy.jsp` - 管理端表单新增景区选择
- 在 `#strategy-modal` 表单中添加景区下拉框（name=`scenicSpotId`）
- `openStrategyEdit(id)` 编辑时预选当前关联景区

## 五、边界情况处理
- 攻略未关联景区（`scenicSpotId=null`）：攻略正常显示在攻略列表页，但不显示在任何景区详情页
- 景区被删除：外键 `ON DELETE SET NULL`，攻略保留但不再显示关联
- 景区详情页无攻略：显示友好提示文案
- 用户未登录发布攻略：保持现有的登录检查逻辑不变
